# 多智能体系统架构

## 架构图

```
用户消息 → 云壹网关 (OpenClaw)
              ↓
        智能路由
              ↓
    ┌─────────┼─────────┐
    ↓         ↓         ↓
   云壹       云捌      其他
  (CEO)   (项目/产品)  (设计/开发)
  Kimi     GLM-4.7    各用各的
    ↑         ↑         ↑
    └─────────┴─────────┘
              ↓
         共享记忆层
```

## 角色定义

| 角色 | 模型 | 职责 | 触发方式 |
|------|------|------|---------|
| **云壹** | Kimi K2.5 | CEO决策、网关路由、统筹协调 | 默认、直接对话 |
| **云捌** | GLM-4.7 | 项目管理、产品分析、风险评估 | @云捌、关键词匹配 |
| **云肆** | Kimi + 万相 | UI设计、视觉创作 | @云肆 |
| **云伍** | Kimi | 前端开发 | @云伍 |
| **云陆** | Kimi | 后端开发 | @云陆 |
| **云柒** | Kimi | 测试质量 | @云柒 |

## 使用方式

### 1. 直接对话（云壹处理）
```
用户: 分析一下市场趋势
云壹: 我来分析...
```

### 2. @云捌（项目管理/产品分析）
```
用户: @云捌 制定项目计划
云壹: 【云捌回复】
      项目计划如下：...
```

### 3. 关键词自动路由
```
用户: 帮我写个PRD
云壹: 【云捌回复】
      PRD大纲：...
```

## 文件结构

```
agents/
├── gateway.js           # 云壹网关 - 智能路由
├── yunba/
│   ├── agent.js         # 云捌智能体 - GLM-4.7
│   └── README.md        # 云捌说明
└── README.md            # 本文件
```

## 扩展新智能体

1. 创建 `agents/yunsi/agent.js`
2. 在 `gateway.js` 中注册
3. 添加路由规则

## 身份与审计

- 每个智能体有独立身份
- 操作以各自名义执行
- 所有操作记录到共享记忆
- 关键操作抄送云壹
