## 二、发现的问题

### 2.1 🔴 严重问题（需立即修复）

#### 问题1: 支付回调缺少签名验证
**位置**: `PaymentService.wxPayCallback()`
```java
@PostMapping("/callback")
public Result<Void> wxPayCallback(@RequestBody Map<String, String> params) {
    // 直接处理回调，未验证签名！
    return paymentService.wxPayCallback(orderNo, wxPayTradeNo, status);
}
```
**风险**: 攻击者可伪造支付回调，篡改订单状态  
**修复**: 添加微信支付签名验证

#### 问题2: 退款接口缺少权限校验
**位置**: `PaymentController.refund()`
```java
@PostMapping("/order/{orderNo}/refund")
public Result<Void> refund(...) {
    // 只验证了登录，未验证是否是订单所有者
    return paymentService.refund(orderNo, reason);
}
```
**风险**: 用户可退款他人订单  
**修复**: 验证订单归属

#### 问题3: 消息内容未过滤
**位置**: `MessageService.sendMessage()`
```java
message.setContent(content); // 直接存储，未过滤
```
**风险**: XSS攻击、敏感词传播  
**修复**: 添加内容过滤和敏感词检测

### 2.2 🟡 中等问题（建议修复）

#### 问题4: 评价提交未验证活动参与
**位置**: `ReviewService.submitReview()`
```java
public Result<Long> submitReview(Review review) {
    // 未验证用户是否参与了该活动
    reviewRepository.insert(review);
}
```
**风险**: 未参与者可随意评价  
**修复**: 验证活动参与记录

#### 问题5: 并发报名存在竞态条件
**位置**: `ActivityService.joinActivity()`
```java
// 检查人数
if (activity.getCurrentParticipants() >= activity.getMaxParticipants()) {
    return Result.error("活动已满员");
}
// 插入记录（非原子操作，存在并发问题）
participantRepository.insert(participant);
```
**风险**: 超卖问题（报名人数超过限制）  
**修复**: 使用数据库乐观锁或分布式锁

#### 问题6: 附近的人查询未限制返回数量
**位置**: `MatchService.getNearbyUsers()`
```java
// Redis查询限制pageSize，但后续数据库查询未限制
List<UserLocation> candidates = userLocationRepository.findCandidates(...);
```
**风险**: 可能返回大量数据，造成内存溢出  
**修复**: 严格限制返回数量

#### 问题7: JWT Token无法主动失效
**位置**: `JwtUtil`
```java
// token一旦签发，直到过期前都有效
// 缺少黑名单机制
```
**风险**: 用户退出后token仍可用，被盗后无法撤销  
**修复**: 添加Redis黑名单机制

### 2.3 🟢 轻微问题（可后续优化）

#### 问题8: 缺少接口限流
**影响**: 可能遭受暴力破解、爬虫攻击  
**建议**: 添加RateLimiter

#### 问题9: 日志记录敏感信息
**位置**: 部分Service日志
```java
log.info("消息发送: from={}, to={}, content={}", senderId, receiverId, content);
// content可能包含敏感信息
```
**建议**: 敏感信息脱敏

#### 问题10: 缺少健康检查端点
**影响**: 无法监控服务状态  
**建议**: 添加/actuator/health

#### 问题11: 数据库连接池配置单一
**位置**: 各服务application.yml
```yaml
hikari:
  maximum-pool-size: 20  # 未区分读写
```
**建议**: 生产环境配置读写分离
