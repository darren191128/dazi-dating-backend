## 1. 项目概述

### 1.1 项目背景
开州通综合服务平台是面向重庆市开州区的一站式本地生活服务平台，整合外卖、商城购物、企业直采、金厨培训等多元业务，为区域内消费者、商家、供应商、采购企业和培训机构提供数字化服务。

### 1.2 项目目标
- **一期目标**: 交付"可交易、可运营、可审计、可扩展"的最小闭环版本
- **核心业务**: 外卖、商城、直采、金厨培训四大业务线
- **用户覆盖**: 消费者、商家、供应商、采购方、金厨学校、平台运营六类角色

### 1.3 业务范围
| 业务线 | 说明 | 一期范围 |
|--------|------|----------|
| 外卖 | 本地餐饮外卖配送 | ✅ 包含 |
| 商城 | 本地电商购物 | ✅ 包含 |
| 直采 | 企业间批量采购 | ✅ 包含 |
| 金厨培训 | 厨师技能培训课程 | ✅ 包含 |
| 酒店预订 | 酒店房间预订 | ❌ 暂缓 |
| 景点门票 | 景区门票销售 | ❌ 暂缓 |
| 研学活动 | 研学旅行服务 | ❌ 暂缓 |
| 信用金融 | 授信、贷款、还款 | ❌ 暂缓 |

## 2. 用户角色定义

### 2.1 消费者 (Consumer)
- **定义**: 平台普通用户，进行外卖点餐、商城购物
- **核心需求**: 浏览商品、下单支付、查看订单、收货评价
- **登录方式**: 手机号验证码 / 微信登录

### 2.2 商家 (Merchant)
- **外卖商家**: 提供餐饮外卖服务
- **商城商家**: 提供商品零售服务
- **核心需求**: 店铺管理、商品维护、订单处理、财务结算

### 2.3 供应商 (Supplier)
- **定义**: 为采购方提供批量商品供应
- **核心需求**: 产品发布、采购订单处理、发货管理、提现结算

### 2.4 采购方 (Buyer)
- **定义**: 企业用户，进行批量采购
- **核心需求**: 企业资质认证、直采商城浏览、批量下单、发票管理

### 2.5 金厨学校 (Training School)
- **定义**: 提供厨师技能培训的机构
- **核心需求**: 课程发布、学员管理、考核认证、财务统计

### 2.6 平台运营 (Platform Operator)
- **定义**: 平台内部运营人员
- **角色细分**: 超级管理员、运营专员、审核员、财务、风控、客服
- **核心需求**: 用户管理、商家审核、订单查询、内容配置、数据监控

## 6. API接口设计概要

### 6.1 服务划分
| 服务 | 职责 | 核心接口 |
|------|------|----------|
| user-service | 用户服务 | 登录注册、用户信息、地址管理 |
| merchant-service | 商家服务 | 入驻审核、商家管理 |
| product-service | 商品服务 | 商品管理、库存管理 |
| order-service | 订单服务 | 订单创建、状态管理、售后 |
| payment-service | 支付服务 | 支付下单、回调、退款 |
| settlement-service | 结算服务 | 结算单、账单、提现 |
| iam-service | 权限服务 | RBAC、菜单、操作日志 |
| content-service | 内容服务 | Banner、公告 |
| training-service | 培训服务 | 课程、报名 |
| project-service | 项目服务 | 项目展示、导入 |
| gateway-service | 网关服务 | 统一鉴权、限流 |

### 6.2 统一状态码

#### 订单主状态
| 状态码 | 含义 | 说明 |
|--------|------|------|
| 10 | 待支付 | 订单已创建，等待支付 |
| 20 | 已支付 | 支付成功，等待履约 |
| 30 | 待接单 | 商家等待确认 |
| 40 | 待履约 | 待取餐/待发货 |
| 50 | 履约中 | 配送中/发货中 |
| 60 | 待完成 | 已送达/已签收 |
| 70 | 已完成 | 业务闭环完成 |
| 80 | 售后中 | 退款或售后处理中 |
| 99 | 已取消 | 取消或退款关闭 |

#### 支付状态
| 状态码 | 含义 |
|--------|------|
| 10 | 待支付 |
| 20 | 支付成功 |
| 30 | 支付失败 |
| 40 | 退款中 |
| 50 | 已退款 |

### 6.3 接口设计原则
- 统一状态机由后端控制，前端只读取状态码
- 敏感操作必须幂等（支付回调、退款、结算）
- 数据查询必须带归属字段过滤
- 分页查询默认 pageSize = 20，最大 100

## 7. 非功能需求

### 7.1 性能需求
| 指标 | 目标值 | 说明 |
|------|--------|------|
| 首页加载 | < 2s | 首屏渲染完成 |
| 接口响应 | < 500ms | P95响应时间 |
| 并发支持 | 1000 QPS | 一期目标 |
| 数据库连接 | < 100 | 单服务连接池 |

### 7.2 安全需求
- **登录安全**: 防暴力破解、短信防刷、异常IP检测
- **数据安全**: 敏感字段加密存储（手机号、身份证）
- **传输安全**: HTTPS全站加密
- **权限安全**: RBAC + 数据范围双重控制
- **审计安全**: 操作日志保留1年以上
- **等保合规**: 满足等保三级要求

### 7.3 可用性需求
- **系统可用性**: 99.9%（年停机时间 < 8.76小时）
- **数据备份**: 每日全量备份 + 实时binlog
- **故障恢复**: RTO < 30分钟，RPO < 5分钟

### 7.4 扩展性需求
- **服务扩展**: 支持独立部署、独立扩容
- **数据扩展**: 表结构预留扩展字段
- **业务扩展**: 配置化业务规则（抽成比例、积分规则）
- **二期预留**: 信用金融、积分体系、分销体系接口预留

### 7.5 技术栈
| 层级 | 技术选型 |
|------|----------|
| 前端 | Vue3 + TypeScript + Element Plus |
| 后端 | Spring Boot + Spring Cloud |
| 数据库 | MySQL 8.0 + Redis |
| 消息队列 | RocketMQ |
| 搜索引擎 | Elasticsearch |
| 任务调度 | XXL-JOB |
| 监控 | Sentinel + Prometheus |
