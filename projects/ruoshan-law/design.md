## 5. 动效设计规范

### 5.1 动效类型

| 动效类型 | 应用场景 | 时长 | 缓动函数 |
|---------|---------|------|---------|
| 淡入 | 元素首次加载 | 0.6s | ease-out |
| 滑入 | 侧边栏、菜单 | 0.4s | cubic-bezier(0.4, 0, 0.2, 1) |
| 悬浮 | 卡片hover | 0.3s | ease-in-out |
| 展开 | 详情展开 | 0.4s | cubic-bezier(0.4, 0, 0.2, 1) |
| 滚动触发 | 视口元素显现 | 0.8s | ease-out |
| 数字递增 | 数据展示 | 1.5s | ease-out |

### 5.2 滚动动效

#### 触发规则
- 使用 Intersection Observer API
- 元素距离视口底部100px时触发
- 一次性动效（仅首次触发）

#### 性能优化
- 使用 CSS transform 和 opacity
- 避免触发重排
- 使用 requestAnimationFrame
- 移动端减少复杂动效

### 5.3 具体动效实现

#### Hero文字动画
```css
.hero-title {
  animation: fadeInUp 1s ease-out forwards;
}

@keyframes fadeInUp {
  from {
    opacity: 0;
    transform: translateY(30px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}
```

#### 卡片悬浮效果
```css
.service-card {
  transition: transform 0.3s ease, box-shadow 0.3s ease;
}

.service-card:hover {
  transform: translateY(-4px);
  box-shadow: 0 8px 24px rgba(0, 0, 0, 0.12);
}
```
