# 重庆启云网络科技有限公司 - 设计规范文档

## 1. 设计理念

### 1.1 设计愿景
以"启云"之名，诠释科技与未来的无限可能。设计应体现：
- **科技感**：前沿、创新、数字化
- **专业可信**：稳重、可靠、值得信赖
- **简约大气**：克制、精致、留白之美

### 1.2 设计原则
1. **清晰优先**：信息层级明确，用户一眼找到所需
2. **克制表达**：少即是多，避免过度装饰
3. **一致体验**：全站视觉语言统一，建立品牌认知
4. **响应优先**：移动端与桌面端同等重视

| **深海蓝** | `#0A1628` | 深色背景、Hero区、页脚 |
| **纯净白** | `#FFFFFF` | 主背景、卡片背景 |

### 2.2 辅助色

| 名称 | 色值 | 用途 |
|-----|------|------|
| **活力青** | `#00D4AA` | 成功状态、数据增长、高亮强调 |
| **智慧紫** | `#6366F1` | 次要强调、标签、图标 |
| **温暖橙** | `#F97316` | 警告提示、重要提醒 |
| **警示红** | `#EF4444` | 错误状态、删除操作 |

### 2.3 中性色阶

| 名称 | 色值 | 用途 |
|-----|------|------|
| **墨黑** | `#0F172A` | 主标题文字 |
| **深灰** | `#334155` | 正文文字 |
| **中灰** | `#64748B` | 次要文字、描述 |
| **浅灰** | `#94A3B8` | 占位符、禁用状态 |
| **银灰** | `#CBD5E1` | 边框、分割线 |
| **雾灰** | `#F1F5F9` | 区块背景、表单边框 |
| **雪白** | `#F8FAFC` | 交替背景、卡片悬停 |

### 2.4 渐变色

```css
/* 主渐变 - Hero背景 */
gradient-primary: linear-gradient(135deg, #0A1628 0%, #1E3A5F 50%, #0A1628 100%);

/* 科技光晕 */
gradient-glow: linear-gradient(90deg, #0066FF 0%, #00D4AA 100%);

/* 卡片微光 */
gradient-card: linear-gradient(180deg, rgba(255,255,255,0.1) 0%, rgba(255,255,255,0) 100%);
```

### 2.5 色彩使用规范

**背景色使用：**
- 深色背景（#0A1628）：Hero区、页脚、强调区块
- 浅色背景（#FFFFFF）：主内容区、卡片
- 交替背景（#F8FAFC）：长页面视觉分隔

**文字色使用：**
- 深色背景上：白色（#FFFFFF）主文字，银灰（#CBD5E1）次要文字
- 浅色背景上：墨黑（#0F172A）主文字，深灰（#334155）正文

**按钮色使用：**
- 主按钮：科技蓝（#0066FF）背景 + 白色文字
- 次要按钮：透明背景 + 科技蓝边框 + 科技蓝文字
- 幽灵按钮：透明背景 + 白色边框（深色背景上）

## 3. 字体规范

### 3.1 字体家族

```css
/* 中文标题字体 */
--font-heading-zh: "PingFang SC", "Hiragino Sans GB", "Microsoft YaHei", sans-serif;

/* 中文正文字体 */
--font-body-zh: "PingFang SC", "Microsoft YaHei", "Hiragino Sans GB", sans-serif;

/* 英文/数字字体 */
--font-en: "Inter", "SF Pro Display", -apple-system, BlinkMacSystemFont, sans-serif;

/* 代码字体 */
--font-mono: "JetBrains Mono", "Fira Code", "SF Mono", monospace;
```

### 3.2 字体层级

| 层级 | 尺寸 | 字重 | 行高 | 字间距 | 用途 |
|-----|------|-----|------|-------|------|
| **Display** | 56px / 3.5rem | 700 | 1.1 | -0.02em | Hero主标题 |
| **H1** | 40px / 2.5rem | 700 | 1.2 | -0.01em | 页面大标题 |
| **H2** | 32px / 2rem | 600 | 1.3 | 0 | 区块标题 |
| **H3** | 24px / 1.5rem | 600 | 1.4 | 0 | 卡片标题 |
| **H4** | 20px / 1.25rem | 600 | 1.4 | 0 | 小标题 |
| **H5** | 18px / 1.125rem | 500 | 1.5 | 0 | 列表标题 |
| **Body Large** | 18px / 1.125rem | 400 | 1.7 | 0.01em | 引导段落 |
| **Body** | 16px / 1rem | 400 | 1.7 | 0.01em | 正文内容 |
| **Body Small** | 14px / 0.875rem | 400 | 1.6 | 0.01em | 辅助说明 |
| **Caption** | 12px / 0.75rem | 400 | 1.5 | 0.02em | 标签、时间 |

### 3.3 响应式字体

| 层级 | 桌面端 | 平板端 | 移动端 |
|-----|-------|-------|-------|
| Display | 56px | 44px | 32px |
| H1 | 40px | 32px | 28px |
| H2 | 32px | 28px | 24px |
| H3 | 24px | 22px | 20px |
| H4 | 20px | 18px | 16px |

### 3.4 字体样式规范

**标题样式：**
- 中文标题避免使用 font-weight: 300（过细影响可读性）
- 大标题可适当使用负字间距增强紧凑感
- 避免大写中文，保持正常书写

**正文样式：**
- 正文行高保持 1.6-1.8，确保阅读舒适
- 段落间距为字体大小的 1.5 倍
- 中文段落首行不缩进，段落间留白分隔

**特殊样式：**
- 强调文字：使用科技蓝（#0066FF）或加粗
- 链接文字：科技蓝 + hover时下划线
- 代码文字：使用等宽字体，浅灰背景

## 4. 间距系统

以 4px 为基础单位，建立 8 点栅格系统：

| Token | 值 | 用途 |
|-------|-----|------|
| space-1 | 4px | 图标间距、紧凑内边距 |
| space-2 | 8px | 小间距、行内元素 |
| space-3 | 12px | 表单元素间距 |
| space-4 | 16px | 标准间距、卡片内边距 |
| space-5 | 20px | 中等间距 |
| space-6 | 24px | 组件间距 |
| space-8 | 32px | 区块内元素间距 |
| space-10 | 40px | 大组件间距 |
| space-12 | 48px | 区块内边距 |
| space-16 | 64px | 区块间距（小） |
| space-20 | 80px | 区块间距（中） |
| space-24 | 96px | 区块间距（大） |
| space-32 | 128px | 页面级间距 |

### 4.2 栅格系统

**桌面端（≥1280px）：**
- 12列栅格
- 列宽：自动计算
- 列间距（Gutter）：24px
- 页面边距：64px
- 最大内容宽度：1200px

**平板端（768px - 1279px）：**
- 8列栅格
- 列间距：20px
- 页面边距：40px

**移动端（<768px）：**
- 4列栅格
- 列间距：16px
- 页面边距：20px

### 4.3 区块间距规范

| 区块类型 | 上内边距 | 下内边距 |
|---------|---------|---------|
| Hero区 | 0 | 96px |
| 标准区块 | 96px | 96px |
| 紧凑区块 | 64px | 64px |
| 页脚 | 64px | 32px |

### 4.4 组件间距规范

**卡片：**
- 内边距：24px（桌面）/ 20px（移动）
- 卡片间距：24px
- 圆角：12px

**按钮：**
- 高度：48px（大）/ 40px（中）/ 32px（小）
- 水平内边距：24px（大）/ 20px（中）/ 16px（小）
- 圆角：8px

**表单输入：**
- 高度：48px
- 水平内边距：16px
- 圆角：8px
- 边框：1px solid #CBD5E1

**导航栏：**
- 高度：72px（桌面）/ 64px（移动）
- 水平边距：64px（桌面）/ 20px（移动）

## 5. 阴影与效果

### 5.1 阴影系统

```css
/* 微阴影 - 卡片默认 */
shadow-sm: 0 1px 2px 0 rgba(0, 0, 0, 0.05);

/* 小阴影 - 按钮悬停、下拉菜单 */
shadow-md: 0 4px 6px -1px rgba(0, 0, 0, 0.1), 0 2px 4px -2px rgba(0, 0, 0, 0.1);

/* 中阴影 - 卡片悬停、模态框 */
shadow-lg: 0 10px 15px -3px rgba(0, 0, 0, 0.1), 0 4px 6px -4px rgba(0, 0, 0, 0.1);

shadow-xl: 0 20px 25px -5px rgba(0, 0, 0, 0.1), 0 8px 10px -6px rgba(0, 0, 0, 0.1);

/* 彩色阴影 - 主按钮 */
shadow-primary: 0 4px 14px 0 rgba(0, 102, 255, 0.39);
```

### 5.2 模糊与 backdrop

```css
/* 导航栏毛玻璃 */
backdrop-blur: blur(12px);
backdrop-bg: rgba(255, 255, 255, 0.8);

/* 深色背景毛玻璃 */
```

### 5.3 光效与渐变


## 7. 动画与过渡

### 7.1 过渡时间

```css
/* 快速反馈 */
duration-fast: 150ms;

/* 标准过渡 */
duration-normal: 250ms;

/* 强调动画 */
duration-slow: 350ms;

/* 页面级动画 */
duration-slower: 500ms;
```

### 7.2 缓动函数

```css
/* 标准 */
ease-default: cubic-bezier(0.4, 0, 0.2, 1);

/* 进入 */
ease-in: cubic-bezier(0.4, 0, 1, 1);

/* 退出 */
ease-out: cubic-bezier(0, 0, 0.2, 1);

/* 弹性 */
ease-bounce: cubic-bezier(0.68, -0.55, 0.265, 1.55);

/* 科技感动画 */
ease-tech: cubic-bezier(0.16, 1, 0.3, 1);
```

### 7.3 常用动画

```css
/* 淡入上移 */
@keyframes fadeInUp {
  from {
    opacity: 0;
    transform: translateY(20px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}

/* 脉冲光晕 */
@keyframes pulse-glow {
  0%, 100% {
    box-shadow: 0 0 20px rgba(0, 102, 255, 0.3);
  }
  50% {
    box-shadow: 0 0 40px rgba(0, 102, 255, 0.6);
  }
}

/* 渐变流动 */
@keyframes gradient-flow {
  0% { background-position: 0% 50%; }
  50% { background-position: 100% 50%; }
  100% { background-position: 0% 50%; }
}
```

## 10. 响应式断点

| 断点 | 宽度 | 设备类型 |
|-----|------|---------|
| sm | 640px | 大手机 |
| md | 768px | 平板竖屏 |
| lg | 1024px | 平板横屏/小笔记本 |
| xl | 1280px | 桌面 |
| 2xl | 1536px | 大桌面 |

### 响应式优先策略

采用 **Mobile-First** 策略：
1. 先设计移动端布局
2. 使用 `min-width` 媒体查询向上扩展
3. 关键内容优先展示
4. 触控友好的交互设计（最小点击区域 44px）

## 11. 可访问性规范

### 11.1 对比度

- 正文文字与背景对比度 ≥ 4.5:1
- 大文字（18px+ 或 14px+ bold）对比度 ≥ 3:1
- 交互元素对比度 ≥ 3:1

### 11.2 焦点状态

- 所有可交互元素必须有可见焦点指示
- 焦点环：2px solid #0066FF，2px 偏移
- 键盘导航顺序符合视觉顺序

### 11.3 语义化

- 正确使用标题层级（h1-h6）
- 表单元素关联 label
- 图片提供 alt 文本
- 使用 ARIA 标签增强可访问性

## 12. 设计文件命名规范

### 12.1 图层命名

```
[类型]/[描述]/[状态]

例如：
- btn/Primary/Default
- btn/Primary/Hover
- card/Feature/Active
- icon/Arrow/Right
- text/Heading/H1
```

### 12.2 组件命名

```
[类别]-[名称]-[变体]

例如：
- Button-Primary
- Button-Primary-Disabled
- Card-Feature
- Input-Text
- Nav-Top
```
