# 子智能体工具调用问题 - 深度诊断报告

## 诊断时间
2026-03-12 16:55

## 问题现象
云伍、云陆等子智能体无法执行 write/edit 操作，任务完成但无实际文件生成。

## 诊断过程

### 1. 日志分析
```
[tools] write failed: Missing required parameter: path (path or file_path)
```
**关键发现**：子智能体调用 write 工具时参数缺失。

### 2. 配置检查
- ✅ 模型配置正常（models.json）
- ✅ API key 配置正确
- ✅ 子智能体认证配置正常
- ❌ 无 tools.json（可能正常，工具由系统提供）
- ⚠️ memory slot plugin not found（doctor 警告）

### 3. 系统状态
- OpenClaw 版本：2026.3.7
- Gateway：运行正常
- 模型：Kimi、Qwen-Plus、GLM-4.7 均可用

## 根因分析

### 最可能原因
**子智能体与模型之间的工具调用协议不匹配**

子智能体运行时：
1. 模型生成工具调用请求
2. 但参数格式不符合 OpenClaw 工具要求
3. 导致 write 调用失败
4. 子智能体任务结束，只返回思考过程

### 技术细节
子智能体的 write 工具需要：
```json
{
  "path": "/absolute/path/to/file",
  "content": "file content"
}
```

但模型可能生成：
```json
{
  "file_path": "/path/to/file",  // 错误参数名
  "content": "..."
}
```

或参数不完整。

## 解决方案

### 方案A：系统级修复（推荐，彻底解决）

**步骤1：更新 OpenClaw**
```bash
openclaw update
# 或重新安装最新版本
```

**步骤2：检查子智能体工具配置**
查看是否有子智能体专用配置需要调整。

**步骤3：测试修复**
创建一个简单的子智能体任务测试 write 功能。

### 方案B：模型参数调优（临时方案）

在子智能体 prompt 中明确指定工具参数格式：
```
使用 write 工具时，必须使用以下格式：
{
  "path": "绝对路径",
  "content": "文件内容"
}
```

### 方案C：绕过子智能体（当前可行）

- 子智能体只负责设计/分析
- 代码生成由我（云壹）在主会话执行
- 使用 write/edit 工具直接创建文件
