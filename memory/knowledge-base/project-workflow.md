# 项目开发流程规范

> 创建时间：2026-03-12
> 适用范围：网站、小程序、APP 等所有项目

## 技术栈
- **前端**：Vue3
- **后端**：Java

## 一个完整网站的组成
- 前台 PC 端 + 手机自适应
- 后台管理系统（CMS）
- 数据库

## 开发流程

### 阶段 1：项目启动
**负责人：云壹（CEO助理）**
- 接收老板任务
- 通知云捌进行项目分析

### 阶段 2：项目分析
**负责人：云捌（项目与产品负责人）**

分析内容必须包含：
1. **前台界面及操作动效**
2. **后台管理系统界面及功能**
3. **数据库设计**
4. **API 接口设计**

输出：整体需求文档

### 阶段 3：UI 设计
**负责人：云肆（UI设计师）**

输出必须包含：
1. **UI 设计标准**
2. **UI 设计图**

完成后通知：云伍、云陆

### 阶段 4：开发准备
**负责人：云伍（前端）、云陆（后端）**

1. 先了解云肆和云捌的输出
2. 审核是否符合需求
3. **如不符合** → 通知云壹统筹，与云捌、云肆校对清楚
4. **确认无误后** → 开始开发

### 阶段 5：开发实现

**云伍（前端工程师）：**
- 前台界面（PC + 手机自适应）
- 后台管理界面（CMS）

**云陆（后端工程师）：**
- 后台功能实现
- 与前台联动的 API
- 数据库实现

### 阶段 6：自检
**负责人：云伍、云陆**

各自检查：
- 是否有功能缺失
- 后台登录后是否可操作
- API 是否正确

## 流程图

```
老板任务 → 云壹 → 云捌[项目分析] → 云肆[UI设计] → 云伍+云陆[审核]
                                              ↓
                                        [不符]→云壹统筹→校对→重新审核
                                              ↓[符合]
                                        云伍[前端] + 云陆[后端]
                                              ↓
                                        各自自检 → 完成
```

## 注意事项

1. **审核环节不可跳过** - 云伍、云陆必须先审核再开发
2. **问题升级机制** - 不符合需求时，必须通过云壹统筹
3. **自检要求** - 开发完成后必须自检，确保功能完整可用
