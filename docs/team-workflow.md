# 七人智能体团队 - 工作统筹机制
# 云壹(CEO助理)作为总负责人，统筹云贰-云柒

## 🏗️ 组织架构

```
                    老板
                     ↓
              云壹 (CEO助理)
             /    |    |    |    |    \
           云贰  云叁  云肆  云伍  云陆  云柒
          (PM)  (PD)  (UI)  (FE)  (BE)  (QA)
```

## 📋 统筹工作流程

### 1. 需求接收与分发

**场景：老板提出需求**
```
老板：开发一个新功能模块
    ↓
云壹接收，分析需求类型
    ↓
判断需要哪些角色参与
    ↓
依次调用子代理处理
```

### 2. 多角色协作流程

**示例：新功能开发**

```
阶段1: 需求分析
  云壹 → 云叁(产品经理)
  输出: PRD文档、用户故事

阶段2: 项目规划  
  云壹 → 云贰(项目经理)
  输出: 项目计划、排期、资源分配

阶段3: 设计阶段
  云壹 → 云肆(UI设计师)
  输出: 设计稿、交互原型

阶段4: 开发阶段
  云壹 → 云伍(前端) + 云陆(后端)
  输出: 代码实现、API接口

阶段5: 测试阶段
  云壹 → 云柒(测试工程师)
  输出: 测试报告、Bug清单

阶段6: 交付
  云壹汇总所有成果，向老板汇报
```

### 3. 触发机制

| 触发方式 | 示例 | 响应角色 |
|---------|------|---------|
| @角色名 | @产品经理 写个PRD | 对应角色 |
| 关键词 | 这个界面怎么设计？ | 云肆(UI) |
| 关键词 | 后端API怎么设计？ | 云陆(后端) |
| 关键词 | 项目进度如何？ | 云贰(PM) |
| 复杂需求 | 开发新功能 | 云壹统筹，多角色协作 |

### 4. 实际工作示例

**示例1：简单咨询**
```
老板: @产品经理 这个需求合理吗？
    ↓
云壹识别 → 创建云叁子代理
    ↓
云叁: 从用户角度分析，建议...
```

**示例2：复杂项目**
```
老板: 开发一个电商支付模块
    ↓
云壹分析: 需要产品+设计+前后端+测试
    ↓
云壹 → 云叁: 输出PRD
云壹 → 云肆: 输出设计稿  
云壹 → 云伍+云陆: 开发实现
云壹 → 云柒: 测试验收
    ↓
云壹汇总: 项目完成，交付物如下...
```

**示例3：问题排查**
```
老板: 系统有点慢，看看什么问题
    ↓
云壹分析: 可能是后端性能问题
    ↓
云壹 → 云陆: 分析系统瓶颈
云陆: 发现数据库查询慢，建议优化索引
    ↓
云壹 → 云伍: 前端是否需要优化？
云伍: 可以懒加载，减少首屏请求
    ↓
云壹汇总: 问题原因+解决方案
```

## 🎯 云壹的核心职责

1. **需求理解** - 准确理解老板意图
2. **任务分解** - 将复杂任务拆解为子任务
3. **角色调度** - 按需调用对应子代理
4. **进度跟踪** - 监控各角色任务进度
5. **结果汇总** - 整合各角色输出，统一汇报
6. **质量控制** - 确保交付物符合要求

## 💡 使用建议

**对老板：**
- 直接提出需求，像对真人团队一样
- 可以@特定角色，也可以让云壹自动分配
- 复杂项目先和云壹沟通整体规划

**对团队：**
- 各角色专注自己领域
- 云壹负责协调，避免信息混乱
- 关键决策点云壹会请示老板

## 📊 优势

1. **一站式入口** - 老板只需对接云壹
2. **专业分工** - 各角色专业领域深耕
3. **高效协作** - 云壹自动协调，无需人工调度
4. **完整闭环** - 从需求到交付，全程覆盖
