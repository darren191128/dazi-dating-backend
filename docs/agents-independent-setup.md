# 七人智能体团队 - 独立飞书配置说明

## 🏗️ 架构设计

每个智能体拥有**独立的飞书应用**，完全隔离运行：

```
云壹 (CEO助理)  →  飞书应用A  →  独立 Gateway (端口18789)
云贰 (项目经理)  →  飞书应用B  →  独立 Gateway (端口18790)
云叁 (产品经理)  →  飞书应用C  →  独立 Gateway (端口18791)
云肆 (UI设计师)  →  飞书应用D  →  独立 Gateway (端口18792)
云伍 (前端工程师) →  飞书应用E  →  独立 Gateway (端口18793)
云陆 (后端工程师) →  飞书应用F  →  独立 Gateway (端口18794)
云柒 (测试工程师) →  飞书应用G  →  独立 Gateway (端口18795)
```

## 📁 配置文件

### 1. 智能体配置
**文件**: `config/agents-feishu.yaml`

定义每个智能体的角色和飞书配置（使用环境变量）。

### 2. 环境变量模板
**文件**: `config/.env.template`

复制为 `.env` 并填入真实的飞书应用凭证：

```bash
cp config/.env.template config/.env
# 编辑 config/.env 填入凭证
```

### 3. 启动器
**文件**: `utils/agent_launcher.py`

用于生成每个智能体的独立配置并启动。

## 🔧 配置步骤

### 第一步：创建飞书应用

在飞书开发者后台为每个角色创建独立应用：

1. 登录 [飞书开发者后台](https://open.feishu.cn/app)
2. 创建企业自建应用
3. 为每个智能体创建一个应用：
   - 云壹-CEO助理
   - 云贰-项目经理
   - 云叁-产品经理
   - 云肆-UI设计师
   - 云伍-前端工程师
   - 云陆-后端工程师
   - 云柒-测试工程师

### 第二步：配置环境变量

```bash
# 复制模板
cp config/.env.template config/.env

# 编辑填入凭证
nano config/.env
```

填入每个应用的 `App ID` 和 `App Secret`：

```env
FEISHU_YUNYI_APP_ID=cli_xxx
FEISHU_YUNYI_APP_SECRET=xxx

FEISHU_YUNER_APP_ID=cli_xxx
FEISHU_YUNER_APP_SECRET=xxx

# ... 其他角色
```

### 第三步：生成配置

```bash
# 生成所有智能体配置
python utils/agent_launcher.py all

# 或单独生成某个智能体
python utils/agent_launcher.py 云贰
```

### 第四步：启动智能体

每个智能体需要独立启动：

```bash
# 方式1: 使用 OpenClaw CLI
OPENCLAW_CONFIG_DIR=/root/.openclaw/agents/云壹 openclaw gateway start
OPENCLAW_CONFIG_DIR=/root/.openclaw/agents/云贰 openclaw gateway start
# ... 其他角色

# 方式2: 使用 systemd (推荐生产环境)
# 为每个智能体创建 systemd 服务
```

## 🎯 使用方式

### 群组绑定

每个飞书应用可以加入不同的群组：

- **云壹** → 综合群、老板私聊
- **云贰** → 项目管理群
- **云叁** → 产品需求群
- **云肆** → 设计评审群
- **云伍** → 前端开发群
- **云陆** → 后端开发群
- **云柒** → 测试质量群

### 消息路由

用户在不同群组中 @对应角色的机器人：

```
用户 @云叁-产品经理: 这个需求怎么设计？
    ↓
云叁 (独立实例) 接收消息
    ↓
以产品经理身份回复
```

## ✅ 优势

1. **完全隔离** - 每个智能体独立运行，互不干扰
2. **独立配置** - 每个角色有自己的飞书应用和权限
3. **灵活扩展** - 可以单独启停某个智能体
4. **安全隔离** - 一个智能体出问题不影响其他

## ⚠️ 注意事项

1. **需要7个飞书应用** - 每个角色一个
2. **端口不能冲突** - 自动分配 18789-18795
3. **内存占用** - 7个实例同时运行需要更多资源
4. **管理复杂度** - 需要分别管理7个应用

## 🔄 替代方案

如果7个独立应用太复杂，也可以考虑：

**方案B: 单应用 + 角色切换**
- 使用1个飞书应用
- 通过消息前缀或@方式切换角色
- 例如: `@产品经理 分析需求`
