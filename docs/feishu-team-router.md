## 🔧 配置步骤

### 1. 获取真实群组 chat_id

在飞书群组中发送消息，查看日志获取 `chat_id`：
```bash
tail -f /tmp/openclaw/openclaw-2026-03-11.log | grep "received message"
```

### 2. 更新路由配置

编辑 `config/feishu-routes.yaml`，将占位符替换为真实 chat_id：

```yaml
routes:
  - chat_id: "oc_xxxxxxxxxxxxxxxx"  # 替换为真实chat_id
    agent: "云贰"
    role: "项目经理"
    ...
```

### 3. 配置群组白名单

在 `openclaw.json` 中添加群组到 `groupAllowFrom`：

```json
"channels": {
  "feishu": {
    ...
    "groupAllowFrom": ["oc_xxx", "oc_xxx", ...]  // 所有群组chat_id
  }
}
```

### 4. 重启 Gateway

```bash
openclaw gateway restart
```

## 🔄 工作流程

```
飞书群组消息
    ↓
云壹 (CEO助理) 接收并识别群组
    ↓
根据 chat_id 路由到对应角色
    ↓
创建子代理 (云贰/云叁/云肆/云伍/云陆/云柒)
    ↓
子代理以专业身份处理消息
    ↓
返回结果 → 云壹汇总 → 回复飞书
```

## 📝 角色分工

### 云壹 - CEO助理 (默认)
- **职责**：统筹全局、协调资源、决策支持
- **特点**：综合性强，跨领域协调
- **触发**：未匹配到特定群组的全部消息

### 云贰 - 项目经理
- **职责**：项目规划、进度跟踪、风险控制
- **特点**：结构化思维，关注时间节点
- **触发**：项目管理群 (oc_pm)

### 云叁 - 产品经理
- **职责**：需求分析、产品设计、用户研究
- **特点**：用户视角，数据驱动
- **触发**：产品讨论群 (oc_product)

### 云肆 - UI设计师
- **职责**：界面设计、交互设计、视觉规范
- **特点**：注重美感与体验
- **触发**：设计评审群 (oc_design)

### 云伍 - 前端工程师
- **职责**：前端开发、组件封装、性能优化
- **特点**：技术细节清晰，代码示例丰富
- **触发**：前端开发群 (oc_frontend)

### 云陆 - 后端工程师
- **职责**：架构设计、API开发、数据库设计
- **特点**：架构思维，关注扩展性
- **触发**：后端开发群 (oc_backend)

### 云柒 - 测试工程师
- **职责**：测试设计、自动化测试、质量保障
- **特点**：严谨细致，关注边界情况
- **触发**：测试质量群 (oc_qa)

```

    ↓
云壹 (CEO助理) 回复：协调云贰查询后，当前进度...
```

## ⚠️ 注意事项

1. **群组必须先添加到白名单** - 否则消息会被静默丢弃
2. **chat_id 区分大小写** - 确保配置与实际一致
3. **子代理自动清理** - 任务完成后自动退出，不占用资源
4. **保持会话可选** - 如需连续对话，可修改 mode 为 "session"
